Layouts e Posicionamento

Aprenda a criar layouts responsivos e controlar o posicionamento de elementos em HTML e CSS, desde técnicas modernas com Flexbox e Grid até posicionamento absoluto e sticky.

Este guia cobre tudo que você precisa saber para estruturar páginas profissionais, desde layouts móveis até desktop, e dominar todos os tipos de posicionamento CSS para criar interfaces complexas e responsivas.

Introdução

Layouts e posicionamento são os pilares da arquitetura visual de qualquer página web. Eles determinam como os elementos se organizam, interagem e se adaptam a diferentes tamanhos de tela. Sem domínio sobre layouts, é impossível criar sites profissionais e responsivos.


Este guia define o padrão oficial de layouts do projeto, cobrindo desde técnicas modernas como Flexbox e Grid até posicionamento (static, relative, absolute, fixed, sticky). O objetivo é que você entenda como estruturar páginas visualmente complexas usando apenas HTML e CSS.


O ponto fundamental: Layouts responsivos não são um "truque de CSS". São uma metodologia estruturada de pensar sobre como elementos se comportam em diferentes contextos.

Objetivos

Este guia estabelece os pilares para que todos os layouts do projeto sejam flexíveis, adaptáveis e acessíveis. Você aprenderá a:


  • Flexbox — Criar layouts unidimensionais com alinhamento preciso

  • Grid — Estruturar layouts bidimensionais complexos

  • Posicionamento — Controlar exatamente onde os elementos aparecem na página

  • Responsividade — Adaptar layouts a diferentes tamanhos de tela com media queries

  • Overflow e containment — Gerenciar espaço e evitar abarrotamento

O que é layout?

Um layout é a organização visual dos elementos em uma página. Define espaçamento, alinhamento, tamanho e comportamento dos blocos de conteúdo. Um bom layout é:


  • Hierárquico — Guia visualmente o usuário pelo conteúdo

  • Equilibrado — Distribui elementos de forma proporcionada

  • Flexível — Adapta-se a diferentes dispositivos sem quebrar

  • Acessível — Mantém leitura lógica e estruturado


Métodos de layout em CSS:


  • Display block/inline — Método tradicional (fluxo normal)

  • Flexbox — Layout unidimensional (uma direção)

  • Grid — Layout bidimensional (linhas + colunas)

  • Posicionamento — Controle absoluto sobre posição

Flexbox (flexible box layout)

Flexbox é um modelo de layout unidimensional que permite organizar elementos em linhas ou colunas com alinhamento automático e distribuição inteligente de espaço. É ideal para componentes, barras de navegação, menus e qualquer layout que precisa se adaptar.


Conceitos fundamentais:


  • Flex container — Elemento pai com display: flex

  • Flex items — Filhos diretos do container

  • Main axis — Eixo principal (horizontal por padrão)

  • Cross axis — Eixo transversal (vertical por padrão)


Estrutura básica:

<div class="flex-container">
    <div class="flex-item">Item 1</div>
    <div class="flex-item">Item 2</div>
    <div class="flex-item">Item 3</div>
</div>

CSS para criar o flex container:

.flex-container {
    display: flex;
    flex-direction: row;              /* row, column, row-reverse, column-reverse */
    justify-content: space-between;  /* Alinha no eixo principal */
    align-items: center;           /* Alinha no eixo transversal */
    gap: 1rem;                   /* Espaço entre items */
}

Propriedades principais do container:



justify-content
Alinha items no eixo principal. Valores: flex-start, flex-end, center, space-between, space-around, space-evenly.

align-items
Alinha items no eixo transversal. Valores: flex-start, flex-end, center, stretch, baseline.

flex-direction
Define se items são exibidos em linha (row) ou coluna (column). Também há versões *-reverse para inverter ordem.

flex-wrap
Define se items quebram em múltiplas linhas. Valores: nowrap (padrão), wrap, wrap-reverse.

gap
Define espaçamento entre items. Substitui margin para um controle mais limpo.

Propriedades dos flex items:


.flex-item {
    flex: 1;                    /* Atalho: flex-grow flex-shrink flex-basis */
    flex-grow: 1;               /* Cresce para preench espaço */
    flex-shrink: 1;             /* Encolhe se necessário */
    flex-basis: 200px;           /* Tamanho base antes de crescer/encolher */
    align-self: center;        /* Override align-items para este item */
}

Dica importante: flex: 1 é a forma mais comum. Significa que o item cresce para preencher espaço disponível de forma igual aos outros items com o mesmo valor.

CSS Grid (para layouts complexos)

Grid é um modelo de layout bidimensional que define estruturas com linhas e colunas. Diferente de Flexbox (que é unidimensional), Grid permite controlar tanto eixos simultaneamente, ideal para layouts de página inteira.


Estrutura básica:

<div class="grid-container">
    <header>Header</header>
    <aside>Sidebar</aside>
    <main>Main</main>
    <footer>Footer</footer>
</div>

CSS para criar o grid container:

.grid-container {
    display: grid;
    grid-template-columns: 200px 1fr 200px;      /* 3 colunas: fixa, flexível, fixa */
    grid-template-rows: auto 1fr auto;         /* header, main, footer */
    gap: 1rem;                          /* Espaço entre items */
    grid-template-areas: 
        "header header header"
        "sidebar main sidebar-right"
        "footer footer footer";
}

Unidades do Grid:


  • fr — "Fração livre" (proporcional). 1fr 2fr = 1ª coluna ocupa 1/3, 2ª ocupa 2/3.

  • auto — Ajusta-se ao conteúdo

  • Medidas fixas — 200px, 10rem, etc.

  • minmax() — Define mínimo e máximo. Ex: minmax(200px, 1fr)


Exemplo de layout com grid-template-areas:

header { grid-area: header; }
aside { grid-area: sidebar; }
main { grid-area: main; }
footer { grid-area: footer; }

Quando usar Grid vs Flexbox: Use Flexbox para componentes simples (navbars, botões). Use Grid para layouts de página inteira ou estruturas com linhas e colunas.

Responsividade com Media Queries

Responsividade móvel garante que uma página se ajuste corretamente em smartphones, tablets e desktops. Usamos media queries em CSS para aplicar estilos diferentes baseado no tamanho da tela.


Abordagem Mobile First (RECOMENDADA):

Comece pelo layout móvel (padrão) e depois melhore para telas maiores: 

/* Estilos padrão para móvel */
.grid-container {
    display: grid;
    grid-template-columns: 1fr;           /* Uma coluna no móvel */
}

/* Tablet e acima (≥768px) */
@media (min-width: 768px) {
    .grid-container {
        grid-template-columns: 200px 1fr;   /* Sidebar + main */
    }
}

/* Desktop e acima (≥1024px) */
@media (min-width: 1024px) {
    .grid-container {
        grid-template-columns: 200px 1fr 300px;  /* 3 colunas */
    }
}

Breakpoints comuns:


  • Móvel: < 768px (padrão, sem media query)

  • Tablet: 768px — 1023px

  • Desktop: ≥ 1024px

  • Ultra-wide: ≥ 1440px (opcional)


Regra importante: Sempre projete para móvel primeiro. Não é um "extra" — é o padrão. Desktop entra depois.

Posicionamento em CSS

Posicionamento permite controlar exatamente onde um elemento aparece na página. Diferente de layouts (Flexbox/Grid), posicionamento tira o elemento do fluxo normal e o coloca em coordenadas específicas.


Os 5 tipos de posicionamento:


Static (padrão)

Todos os elementos começam com position: static. Significa que o elemento segue o fluxo normal do documento, um após o outro. 

.elemento {
    position: static;  /* Padrão */
    top: 10px;        /* NÃO funciona com static */
}

Características:

  • Propriedades top, right, bottom, left são ignoradas

  • Elemento não sai do fluxo

  • Z-index não funciona


Relative (relativo à posição original)

O elemento é posicionado relativamente à sua posição original no fluxo normal. Todos os outros elementos "fingem" que ele ainda está lá (ocupam espaço). 

.elemento {
    position: relative;
    top: 20px;        /* Move 20px para baixo */
    left: 30px;       /* Move 30px para direita */
}

Características:

  • Posicionado em relação à sua posição original

  • Mantém o espaço no fluxo (outros elementos não se movem)

  • Z-index funciona

  • Ideal como "pai" para elementos com position: absolute

Caso de uso: Criar um "contexto de posicionamento" para elementos filhos absolutos, ou pequenos ajustes de posição.


Absolute (relativo ao pai posicionado)

O elemento é posicionado em relação ao seu pai mais próximo que tem position: relative, absolute, fixed, ou qualquer outro valor que não seja static. Se não houver, usa <html>. 

.pai {
    position: relative;  /* Cria contexto */
}

.filho {
    position: absolute;
    top: 10px;
    left: 20px;
    width: 200px;
}

Características:

  • Sai completamente do fluxo (outros elementos ignoram seu espaço)

  • Coordenadas são relativas ao pai posicionado mais próximo

  • Z-index funciona (controla profundidade)

  • Width/height costumam ser necessários

Casos de uso: Tooltips, dropdowns, badges fixos em cards, sobreposições.


Fixed (relativo à janela)

O elemento é posicionado em relação à janela do navegador. Quando o usuário rola a página, o elemento fica imóvel (fixo). Comum em: navbars, botões flutuantes, sticky headers. 

.navbar {
    position: fixed;
    top: 0;
    left: 0;
    right: 0;
    height: 60px;
    background: var(--primary-color);
    z-index: 999;         /* Fica acima de tudo */
}

Características:

  • Sai do fluxo completamente

  • Coordenadas são relativas à viewport (janela do navegador)

  • Não se move com scroll

  • Z-index funciona (geralmente precisa de valor alto)

Casos de uso: Navbar fixa, botão flutuante "back to top", modais, sidebars flutuantes.


Sticky (entre relative e fixed)

Hybrid entre relative e fixed. O elemento começa no fluxo normal, mas quando o usuário rola até sua posição, ele fica "colado" ao viewport até que saia de seu elemento pai. 

thead {
    position: sticky;
    top: 0;              /* Fica no topo quando rola */
    background: white;    /* Necessário: não quer que conteúdo apareça em cima */
    z-index: 10;
}

Características:

  • Começa no fluxo normal

  • Se torna fixed ao atingir o offset definido (top, left, etc.)

  • Volta ao fluxo quando pai sai da viewport

  • Precisa de top, right, bottom ou left para funcionar

  • Z-index funciona

Casos de uso: Headers de tabela, headings de seção (que viram sticky ao scroll), sidebars que fixam ao atingir topo.


Dica importante: O posicionamento mais comum é absolute (para pequenos ajustes) ou fixed (para navbars). Use relative apenas para criar "contexto" para elementos absolutos filhos. Sticky é uma opção moderna e elegante quando bem aplicado.

Overflow (gerenciando transbordamento)

Quando conteúdo é maior que o container, ele transborda. overflow define como lidar com isso: mostrar scrollbar, cortar, ou esconder.


Valores principais:


.container {
    width: 300px;
    height: 200px;
    overflow: auto;        /* visible, hidden, scroll, auto */
}

  • visible — Padrão. Conteúdo transborda (não corta)

  • hidden — Corta conteúdo que sai do container

  • scroll — Sempre mostra scrollbar (mesmo sem transbordamento)

  • auto — Mostra scrollbar apenas se necessário (RECOMENDADO)


Casos de uso: Dropdowns com muitas opções, modais com conteúdo longo, sidebars. Essencial quando o container tem altura fixa mas o conteúdo é dinâmico.
