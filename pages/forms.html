Este guia define o padrão oficial de formulários do projeto.
Todos os formulários devem seguir estas regras para manter consistência, acessibilidade e validação adequada.

1. Objetivos

Padronizar

  • Estrutura de formulários
  • Tipos de campos
  • Validações nativas
  • Estilos visuais

2. Estrutura base (OBRIGATÓRIO)

Todos os formulários devem seguir essa estrutura

            
<form class="form">
    <div class="form-group">
        <label for="nome">Nome</label>
        <input id="nome" type="text" required>
    </div>
    <button type="submit">Enviar</button>
</form>

Entendendo cada elemento

  • <form>: Contêiner principal do formulário.
  • .form-group: Agrupa um campo e seu rótulo para melhor organização.
  • <label>: Rótulo descritivo para o campo, associado via atributo for.
  • <input>: Campo de entrada de dados, com tipo e validação definidos.
  • <button>: Botão para enviar o formulário.

3. Por que usar <label>?

O label significa "rótulo".

Os rótulos são importantes para acessibilidade e usabilidade. Eles:

  • Facilitam a navegação por teclado.
  • Oferecem contexto aos usuários sobre o que o campo espera.
  • Permitem a seleção de campos ao clicar no rótulo.
            
<label for="nome">Nome</label>
<input id="nome" type="text" required>

4. Tipos de <input> (com explicação)

O elemento <input> muda de comportamento de acordo com o atributo type. Use o tipo certo para cada dado.

🔤 Texto (text)

<input type="text" placeholder="Seu nome">

Usado para:

  • Nome
  • Cidade
  • Qualquer texto curto

📧 Email (email)

<input type="email" required>

O que acontece:

  • O navegador valida automaticamente o formato.
  • Exige @ e domínio válido.

🔒 Senha (password)

<input type="password" minlength="6">

O que acontece:

  • Os caracteres são escondidos.
  • Pode exigir tamanho mínimo.

🔢 Número (number)

<input type="number" min="0" max="10">

Ideal para:

  • Idade
  • Quantidade

🔘 Rádio (radio)

<input type="radio" name="plano" value="basic"> Básico
<input type="radio" name="plano" value="pro"> Pro

Importante:

  • Mesmo name → apenas uma escolha é permitida.

☑️ Checkbox (checkbox)

<input type="checkbox"> Aceito os termos

Permite:

  • Múltiplas seleções (vários checkboxes juntos).
  • Confirmações (ex.: aceite de termos).

5. Select (lista suspensa)

            
<select required>
<option value="">Selecione</option>
<option value="1">Opção 1</option>
</select>

Use quando:

  • Existem várias opções.
  • Quer economizar espaço na tela.

6. Textarea (texto longo)

<textarea rows="4" placeholder="Mensagem"></textarea>

Ideal para:

  • Comentários
  • Mensagens
  • Descrições

7. Botões

<button type="submit">Enviar</button>
<button type="reset">Limpar</button>

Tipos:

  • submit - envia o formulário.
  • reset - limpa os campos.
  • button - ação customizada via JavaScript.

8. Validações nativas (SEM JavaScript)

O HTML já valida automaticamente vários campos. Use essas validações para melhorar a experiência do usuário.

Campo obrigatório

<input required>

Tamanho mínimo

<input minlength="3">

Intervalo numérico

<input type="number" min="1" max="100">

Padrão (regex)

<input pattern="[A-Za-z]{3,}">

Exemplo: Só aceita letras, mínimo 3 caracteres.

9. Feedback visual com CSS

Aqui está o diferencial: o usuário vê imediatamente se está certo ou errado.

Estados de validação

.input:valid {
  border: 2px solid green;
}

.input:invalid {
  border: 2px solid red;
}

Campo em foco

.input:focus {
  border-color: blue;
  outline: none;
}

Por que isso é importante?

  • ✔ O usuário entende erros rapidamente.
  • ✔ Reduz envio incorreto.
  • ✔ Melhora a experiência.

10. Padrão visual recomendado

.form-group {
  display: flex;
  flex-direction: column;
  margin-bottom: 16px;
}

.input {
  padding: 10px;
  border: 1px solid #ccc;
  border-radius: 6px;
}

11. Erros comuns (evite isso)

  • ❌ Não usar <label>.
  • ❌ Usar só placeholder.
  • ❌ Não validar campos.
  • ❌ Inputs sem padrão visual.

12. Boas práticas

  • ✔ Sempre use <label>.
  • ✔ Use validação nativa antes de JavaScript.
  • ✔ Dê feedback visual.
  • ✔ Mantenha padrão de classes.

13. Exemplo completo explicado

<form class="form">
  <div class="form-group">
    <label for="email">Email</label>
    <input class="input" id="email" type="email" required>
  </div>

  <div class="form-group">
    <label for="senha">Senha</label>
    <input class="input" id="senha" type="password" minlength="6" required>
  </div>

  <button type="submit">Entrar</button>
</form>

O que está correto aqui:

  • Campos com <label>.
  • Validação nativa ativa.
  • Estrutura organizada.
  • Pronto para estilização.
